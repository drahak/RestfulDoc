Request content type

API can provide you more then one response type for each resource. If you like to work with XML, just use XML. There available these Content-Types:

  • JSON: application/json
  • JSONP: application/javascript
  • XML: application/xml
  • QUERY: application/x-www-form-urlencoded
  • DATA_URL: application/x-data-url

Example request

GET /v1/resource/name HTTP/1.1
Content-Type: application/xml

Errors

Errors are provided as any other resource in Restful library. Restful uses conventional HTTP response codes to indicate success or failure of an API request. In general, codes in the 2xx range indicate success, codes in the 4xx range indicate an error that resulted from the provided information (e.g. a required parameter was missing, a charge failed, etc.), and codes in the 5xx range indicate an error with API servers.

Common used response codes:

200 OK Response to a successful GET, PUT, PATCH or DELETE. Can also be used for a POST that doesn't result in a creation.
201 Created Response to a POST request which successfully created an entity. There should also be Location header with new entity endpoint.
204 No Content Response every time when response body is empty. API should response ith 204 only in case if you send POST request.
400 Bad Request Response when this request is malformed. For example API could not find correct parser for provided content type.
401 Unauthorized When client is not authorized, authorization header not provided or when request is unsecured (outside of SSL)
403 Forbidden When user is authenticated but has no permission to access this resource.
404 Not Found When non-existing resource is requested.
405 Method Not Allowed When an HTTP method is invalid (does not exist) for given resource name. In this case, API provides list of usable HTTP methods.
410 Gone When resource is no longer available (e.g. was moved or deleted). Error message could contain new resource location if resource still exists but with new name.
415 Unprocessable entity When validation errors appear when requesting resource. Response includes validation errors usualy under the errors key.
429 Too Many Requests When request is rejected due to client reaches rate limit for given time period.
